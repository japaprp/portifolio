Integracao sob controle

Cliente com site proprio nao precisa virar caos tecnico.

O Amorim PDV Mobile nao pede que o cliente abandone o que ja tem. O caminho certo e acoplar por contrato ou adapter, mantendo o app limpo, a venda escalavel e a manutencao sob criterio.

Regra de ouro

  • nao adaptar o app para cada cliente no susto
  • nao apontar o app para API arbitraria
  • nao liberar operacao sem homologacao minima

O que faz sentido

Preservar o core do produto e adaptar so a ponte de integracao. Isso reduz risco, custo e desgaste de suporte no medio prazo.

Contrato food

  • lanchonete, hamburgueria, bar, restaurante
  • pedido, preparo, pagamento e expedicao
  • bom para operacao de salao e cozinha

Contrato varejo

  • mercado, hortfruit, padaria e farmacia
  • catalogo, leitura, checkout e balcao
  • separacao por businessProfile

Adapter

  • entra quando a API do cliente nao bate no contrato
  • protege o app de customizacao desordenada
  • deixa onboarding mais rapido e seguro
Passo 1

Escolher o contrato certo

Primeiro enquadra o negocio em `food` ou `varejo`. Sem isso, a integracao comeca torta e o suporte vira excecao acumulada.

Passo 2

Apontar para adapter ou API homologada

Se a API do cliente ja falar o contrato, conecta. Se nao falar, o adapter faz a ponte. O app continua limpo.

Passo 3

Homologar antes do go-live

Login, busca, operacao, sync e erro. O minimo precisa ser validado antes de liberar uso real.

O que mais reduz dor de cabeca futura

  • usar o businessProfile correto por cliente e setor
  • manter a camada de integracao fora do app quando o sistema for terceiro
  • passar por homologacao curta, mas obrigatoria
  • documentar o combinado de implantacao e suporte desde o inicio
